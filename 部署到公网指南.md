# PPT HTML Studio 本地保活与公网部署指南

## 1. 本地长期运行方案

本项目现在提供一套 Windows 保活脚本：

```text
D:\workspace\ppt-html-platform\app\scripts\start-platform.ps1
D:\workspace\ppt-html-platform\app\scripts\watchdog.ps1
D:\workspace\ppt-html-platform\app\scripts\install-startup-task.ps1
D:\workspace\ppt-html-platform\app\scripts\uninstall-startup-task.ps1
```

### 1.1 手动后台启动

运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File D:\workspace\ppt-html-platform\app\scripts\start-platform.ps1
```

默认地址：

```text
http://127.0.0.1:5177/
```

日志位置：

```text
D:\workspace\ppt-html-platform\app\data\logs\server.out.log
D:\workspace\ppt-html-platform\app\data\logs\server.err.log
```

### 1.2 启动本地看门狗

看门狗会每 30 秒访问：

```text
http://127.0.0.1:5177/api/health
```

如果服务挂了，会自动重新启动。

手动运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File D:\workspace\ppt-html-platform\app\scripts\watchdog.ps1
```

日志位置：

```text
D:\workspace\ppt-html-platform\app\data\logs\watchdog.log
```

### 1.3 注册为 Windows 登录后自动保活

运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File D:\workspace\ppt-html-platform\app\scripts\install-startup-task.ps1
```

它会创建计划任务：

```text
PPTHTMLStudioWatchdog
```

该任务会在 Windows 用户登录后自动启动看门狗。

### 1.4 取消自动保活

运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File D:\workspace\ppt-html-platform\app\scripts\uninstall-startup-task.ps1
```

### 1.5 如果计划任务被系统拒绝

某些 Windows 权限策略会拒绝创建计划任务。可以改用当前用户的启动文件夹快捷方式：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File D:\workspace\ppt-html-platform\app\scripts\install-startup-shortcut.ps1
```

它会在当前用户的 Startup 文件夹里创建快捷方式，并立即启动看门狗。

取消：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File D:\workspace\ppt-html-platform\app\scripts\uninstall-startup-shortcut.ps1
```

### 1.6 本地保活的边界

本地看门狗能处理：

- Python 服务崩溃；
- 端口服务退出；
- Codex/终端关闭导致原前台进程消失；
- Windows 登录后自动启动。

它不能处理：

- 电脑关机；
- Windows 深度睡眠；
- 当前 Windows 用户未登录；
- 网络断开；
- 杀毒软件或系统策略阻止 Python。

如果要做到真正 7x24 小时在线，建议部署到云服务器。

## 2. 局域网内给别人访问

如果只想让同一个 Wi-Fi 或公司内网的人访问，可以把服务绑定到 `0.0.0.0`：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File D:\workspace\ppt-html-platform\app\scripts\start-platform.ps1 -HostName 0.0.0.0 -Port 5177
```

然后查看本机局域网 IP：

```powershell
ipconfig
```

例如本机 IP 是：

```text
192.168.1.23
```

别人访问：

```text
http://192.168.1.23:5177/
```

如果无法访问，需要在 Windows 防火墙放行 TCP `5177` 端口。

## 3. 临时公网访问方案

适合演示、临时分享，不适合长期正式使用。

常见选择：

- Cloudflare Tunnel
- ngrok
- cpolar
- frp

### 3.1 Cloudflare Tunnel 示例

本地服务保持：

```text
http://127.0.0.1:5177
```

启动 tunnel，把公网域名转发到本地：

```powershell
cloudflared tunnel --url http://127.0.0.1:5177
```

Cloudflare 会给一个临时 HTTPS 地址，别人可通过该地址访问。

### 3.2 ngrok 示例

```powershell
ngrok http 5177
```

ngrok 会生成一个公网 HTTPS 地址。

## 4. Vercel 部署方案

项目根目录已经增加 Vercel 所需文件：

```text
D:\workspace\ppt-html-platform\vercel.json
D:\workspace\ppt-html-platform\api\index.py
D:\workspace\ppt-html-platform\requirements.txt
```

上传 GitHub 后，在 Vercel 中导入该仓库，建议配置：

```text
Framework Preset: Other
Root Directory: ./
Build Command: 留空
Output Directory: 留空
Install Command: 留空，或使用 pip install -r requirements.txt
```

部署后访问根域名即可打开平台首页。`vercel.json` 会把：

```text
/                  -> app/frontend/index.html
/static/*          -> app/frontend/static/*
/api/*             -> api/index.py
/outputs/*         -> api/index.py
```

### 4.1 Vercel 版本的重要限制

Vercel 可以解决 404 和页面入口问题，但它不是长期运行的 Python 服务器。当前 Vercel 版本适合演示和小 PPT 测试，需要注意：

- 生成结果、上传文件、API 设置会写入 `/tmp/ppt-html-platform-data`，这是 Vercel 临时目录，实例休眠或切换后可能丢失。
- Vercel Serverless 请求体大小有限，不能像本地一样上传几十 MB 或 100MB PPT。线上页面会提示小文件模式。
- 大 PPT、慢 AI 模型、长时间生成可能触发函数超时。
- 如果需要稳定多人使用，建议接入 Vercel Blob / KV / 数据库，或者改用 Render、Railway、Fly.io、云服务器等长期运行后端的平台。

### 4.2 Vercel 上 API Key 怎么处理

页面中保存 API Key 会暂存在 `/tmp`，不保证长期保存。更稳的方式是：

1. 每次使用前在页面中重新粘贴 key；
2. 或后续改造为环境变量/数据库保存；
3. 不要把 `app/data/integration.json` 上传到 GitHub。

## 5. Cloudflare Pages 部署方案

Cloudflare Pages 的构建日志如果出现：

```text
npm error enoent Could not read package.json
Failed: build command exited with code: 1
```

说明 Cloudflare 被配置成执行 `npm run build`，但仓库根目录没有 `package.json`。项目现在已经补齐：

```text
package.json
scripts/build-cloudflare.mjs
wrangler.toml
functions/api/[[path]].js
functions/outputs/[[path]].js
```

### 5.1 Cloudflare Pages 构建设置

在 Cloudflare Pages 中使用以下配置：

```text
Framework preset: None
Build command: npm run build
Build output directory: dist
Root directory: /
```

`npm run build` 会把：

```text
app/frontend/index.html
app/frontend/static/*
```

复制到：

```text
dist/
```

这样 Cloudflare 就能部署前端页面。

### 5.2 关于后端 API

Cloudflare Pages 不能直接运行当前的 Python `python-pptx` 后端。当前项目增加了 Pages Functions 代理：

```text
/api/*      -> functions/api/[[path]].js
/outputs/*  -> functions/outputs/[[path]].js
```

如果只部署 Cloudflare Pages 而不配置后端，页面可以打开，但上传 PPT、生成 HTML、下载 ZIP 等功能会提示后端未配置。

要完整使用平台，需要先把 Python 后端部署到 Render、Railway、Fly.io、云服务器或本地公网隧道，然后在 Cloudflare Pages 的环境变量中添加：

```text
BACKEND_ORIGIN=https://你的后端域名
```

示例：

```text
BACKEND_ORIGIN=https://ppt-html-backend.onrender.com
```

配置后，Cloudflare Pages 会把 `/api/*` 和 `/outputs/*` 自动转发到这个后端。

### 5.3 推荐架构

```text
Cloudflare Pages：托管前端页面
Render/Railway/Fly/云服务器：运行 Python 后端
BACKEND_ORIGIN：连接前端和后端
```

如果要完全跑在 Cloudflare 内部，需要把 Python 后端重写成 Cloudflare Workers 支持的 JavaScript/TypeScript，并把上传文件、生成结果、API 设置迁移到 R2/KV/D1。这是另一轮较大的重构。

## 6. 正式公网部署方案

推荐使用云服务器，而不是让自己电脑长期暴露在公网。

### 4.1 服务器要求

最低配置：

- 1 核 CPU
- 1GB 到 2GB 内存
- Python 3.10+
- 20GB 磁盘

建议配置：

- 2 核 CPU
- 4GB 内存
- 40GB+ 磁盘
- Ubuntu 22.04 / Debian 12 / Windows Server

### 4.2 复制项目到服务器

把以下目录上传到服务器：

```text
D:\workspace\ppt-html-platform\app
```

注意：

```text
data\integration.json
```

里面可能保存 API key。公开部署前要确认密钥安全。

### 4.3 Linux 服务器启动

进入项目目录：

```bash
cd /opt/ppt-html-platform/app
```

安装依赖：

```bash
python3 -m pip install python-pptx pillow
```

启动：

```bash
python3 backend/server.py --host 127.0.0.1 --port 5177
```

### 4.4 systemd 常驻

创建服务文件：

```bash
sudo nano /etc/systemd/system/ppt-html-platform.service
```

写入：

```ini
[Unit]
Description=PPT HTML Studio
After=network.target

[Service]
WorkingDirectory=/opt/ppt-html-platform/app
ExecStart=/usr/bin/python3 /opt/ppt-html-platform/app/backend/server.py --host 127.0.0.1 --port 5177
Restart=always
RestartSec=3
User=www-data
Environment=PYTHONUNBUFFERED=1

[Install]
WantedBy=multi-user.target
```

启用：

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now ppt-html-platform
sudo systemctl status ppt-html-platform
```

### 4.5 Nginx 反向代理

安装 Nginx：

```bash
sudo apt install nginx
```

站点配置：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    client_max_body_size 120m;

    location / {
        proxy_pass http://127.0.0.1:5177;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

启用 HTTPS：

```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

## 7. 公网安全提醒

当前平台默认没有登录系统。公网开放前，至少做一项保护：

- Nginx Basic Auth；
- Cloudflare Access；
- VPN 内网访问；
- 只允许固定 IP；
- 加一个平台登录页。

不建议无保护直接公开，因为平台支持：

- 上传 PPT；
- 写入服务器文件；
- 保存外部 API key；
- 生成和下载文件。

## 8. 推荐选择

如果只是自己长期使用：

```text
本地看门狗 + Windows 计划任务
```

如果给几个人临时试用：

```text
本地看门狗 + Cloudflare Tunnel / ngrok
```

如果给团队长期使用：

```text
云服务器 + systemd + Nginx + HTTPS + 访问认证
```
